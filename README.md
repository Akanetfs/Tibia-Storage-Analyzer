# Tibia-Storage-Analyzer
Storage analyzer tool

Español:

He creado esta herramienta la cual sirve para detectar storages duplicados dentro del servidor tanto en archivos .lua en actions/creaturescripts/npc/movements/globalevents entre otros como tambien el archivo storage.lua/storages.lua/051-storages.lua para poder detectar mas a fondo storages que puedas estar duplicados.

Como usar:

- Seleccionar la carpeta del servidor.
- Dar a analizar para que muestre los posibles storages duplicado de tu servidor.
- Elegir el tipo de editor para modificar archivos (vs code o notepad++).
- En el caso de que exista un posible storage duplicado dar doble click, lo cual hará que se muestre una ventana en donde muestra el archivo del problema, el storage, la linea del problema.
- Por ultimo para modificar el archivo, dar doble click a la opcion en donde especifica el archivo con el posible storage duplicado.



English:

I have created this tool which is used to detect duplicate storages within the server, both in .lua files in actions/creaturescripts/npc/movements/globalevents, among others, as well as in the storage.lua / storages.lua / 051-storages.lua file. This allows for deeper detection of storages that may be duplicated.

How to use it:

- Select the server folder.
- Click “Analyze” so it displays possible duplicate storages in your server.
- Choose the type of editor to modify files (VS Code or Notepad++).
- If a possible duplicate storage exists, double-click it. This will open a window showing the problematic file, the storage, and the line where the issue occurs.
. Finally, to modify the file, double-click the option that specifies the file containing the possible duplicate storage.


Português:

Criei esta ferramenta que serve para detectar storages duplicados dentro do servidor, tanto em arquivos .lua em actions/creaturescripts/npc/movements/globalevents, entre outros, como também no arquivo storage.lua / storages.lua / 051-storages.lua, permitindo detectar mais a fundo storages que possam estar duplicados.


Como usar:

- Selecionar a pasta do servidor.
- Clicar em “Analisar” para que sejam exibidos os possíveis storages duplicados do seu servidor.
- Escolher o tipo de editor para modificar arquivos (VS Code ou Notepad++).
- Caso exista um possível storage duplicado, dar duplo clique, o que abrirá uma janela mostrando o arquivo do problema, o storage e a linha onde ocorre o problema.
- Por último, para modificar o arquivo, dar duplo clique na opção que especifica o arquivo com o possível storage duplicado.
